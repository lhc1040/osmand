Monitoring (UX changes)

  • The 'Logging Services' top widget was deleted. The functionality is now available via the 'GPX recording' widget, which is visible by default in all application modes when the Logging services plugin is enabled.
  • The 'GPX recording' widget now includes the following actions: 'Start/Stop GPX recording', 'Start new GPX segment', 'Save current GPX track' (does not stop recording), and 'Enable/Disable online tracking' (only if online tracking is configured).
  • The 'GPX recording' widget controls the track recording independent of the use profile (and independent of the 'Logging services' settings), starts/stops the background service (sleep mode) as needed, and restores the state after an application restart.
  • The 'Logging services' settings control track recording for the specific profiles and can not be changed via the 'GPX recording' widget (i.e. where track recording is specifically enabled in the settings, it will still be recorded even if not globally enabled via the widget).
  • The app actions 'Navigation' and 'Start GPX recording' automatically start the 'Sleep mode functionality' (a background service which keeps GPS on while OsmAnd is running in background or while the screen is off). This 'Sleep mode functionality' is also available manually in the 'Logging services' settings.
  • Online tracking now works only while GPX recording is enabled.
